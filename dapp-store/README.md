# Stealf — Solana dApp Store publishing

Workspace to build, sign, and submit Stealf to the Solana dApp Store
(https://publish.solanamobile.com — the store that ships preinstalled on
every Seeker phone).

## TL;DR

```bash
# 1. Once: generate a dedicated keystore (back up the resulting file +
#    passwords in 1Password — losing them locks Stealf out of updates).
./dapp-store/generate-keystore.sh

# 2. Build a signed APK with the dapp-store profile.
eas build --platform android --profile dapp-store
# When prompted for credentials, choose "upload your own" and provide
# stealf-dappstore.keystore + the password you set above.

# 3. First submission only: go to https://publish.solanamobile.com,
#    register, KYC/KYB, mint the App NFT, upload the APK via the UI.

# 4. For every later version: run the wrapper.
export DAPP_STORE_API_KEY=...   # from /dashboard/settings/api-keys
./dapp-store/publish-update.sh ./build/stealf-release.apk ./publisher-keypair.json "Seeker MWA + yield fixes"
```

## Files in this folder

| File | Purpose |
|---|---|
| `generate-keystore.sh` | Creates the dApp-Store-only signing keystore. Run once, back up the result. |
| `publish-update.sh` | Wraps `@solana-mobile/dapp-store-cli` for incremental release uploads. |
| `README.md` | This file. |
| `stealf-dappstore.keystore` | Generated by step 1. Gitignored — back it up out-of-band. |
| `publisher-keypair.json` | The Solana keypair file for your publisher wallet. Gitignored. |

## First-submission checklist

You'll do this once via the Publisher Portal web UI. Have these ready
BEFORE you start so you don't lose your place mid-submission:

### Wallet + admin

- [ ] Browser-extension Solana wallet (Phantom / Solflare / Backpack) loaded with **~0.2 SOL** for fees + storage. This wallet is permanent — it's the only one that can publish updates for Stealf forever.
- [ ] KYC/KYB: legal name + ID for individuals, or company docs for entities. Approval is human-reviewed and can take a couple of days.

### App metadata

- [ ] App name: **Stealf**
- [ ] Short description (≤80 chars): e.g. *"Privacy-first Solana neobank with yield, stealth wallet, and Visa card"*
- [ ] Long description (≤4000 chars): pitch + key features + what makes Stealf different from a standard Solana wallet.
- [ ] Category: **Finance** (sub-category Wallet)
- [ ] Website: https://stealf.xyz
- [ ] Privacy policy URL (mandatory): publish one at e.g. `https://stealf.xyz/privacy` if you don't already have it.
- [ ] Support email: e.g. `support@stealf.xyz`

### Visual assets

| Asset | Spec | Notes |
|---|---|---|
| App icon | **512×512 PNG** | Square, opaque, no transparency at edges. Same logo as Google Play. |
| Feature graphic / banner | **1200×600 PNG** | Wide hero image shown at the top of the listing. |
| Screenshots | **4 to 8 images, 16:9 portrait, min 1080×1920** | One per key feature: home / cash wallet, stealth wallet, yield, send, swap, card. |
| Optional video | YouTube link | A 30s demo helps conversion. |

### Build artefact

- [ ] Signed `.apk` (NOT an `.aab` — Google Play uses AAB but the dApp Store requires APK).
- [ ] Built with the dedicated `stealf-dappstore.keystore` (not the Google Play keystore).
- [ ] `versionCode` higher than any previous dApp-Store submission (auto-incremented by EAS).
- [ ] `targetSdkVersion` ≥ 34 (Android 14, current Play Store baseline).

## Publisher policy quick-check (Stealf-specific)

| Rule | Stealf status |
|---|---|
| Privacy policy reachable from the app + listing | TODO — publish one at stealf.xyz/privacy. Mandatory. |
| Account deletion path inside the app | DONE — Profile → Delete Account. |
| No webviews loading untrusted content | DONE — only Rhino.fi widget (fixed origin). |
| Prohibited content (gambling, drugs, weapons, hate) | DONE — N/A. |
| KYC for fiat / financial features disclosed | DONE — Rain card flow gates on KYC. Mention in long description so reviewers don't flag it. |
| Encryption for sensitive data | DONE — Turnkey + SecureStore. |
| Permissions list ≤ what the features need | Check `app.config.js` Android perms before submitting. |

## After submission

- Review takes **3–5 business days**, feedback sent from `publishersupport@dappstore.solanamobile.com`.
- If rejected: read the feedback, fix, re-submit a higher `versionCode`. The publisher wallet stays the same, the App NFT stays the same.
- Once approved: the listing goes live on every Seeker device's dApp Store.
